Documentation

  • Explanation

    This page documents the development of "Help! I'm Bored!"

  • API Chosen

    Bored API is a fairly straight-forward service. A user submits an http request and the api responds with an activity represented by a JSON object. Each object contains several values which can be used to sort the possible responses. No authentication required.

  • Connecting to Bored API

    In order to successfully connect with Bored API, a proxy was set up using Heroku App. This is done by prepending the API request url with the proxy service. This has caused unreliable loading times when requesting a new activity from Bored API.

  • Functional api search

    You can now (clunk-ily) send a request to Bored API and get a request that is displayed on the website.

  • Email form not functional

    The form for sending an email reminder is there but it has no functionality. As it turns out JS doesn't have that capability on its own

  • Local storage set up for Bored API search

    All values for the api search are stored in localStorage and are displayed once a user revists the site

  • Email reminder abandoned

    Another functionality will need to be implemented.

  • Color scheme and app design completed

    App is basically done.
